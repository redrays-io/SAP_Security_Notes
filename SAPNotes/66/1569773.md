### Title
**1569773 - Security sessions might remain alive after expiration period**

### Description
#### Symptom
In some scenarios, HTTP application sessions might remain alive after the expiration of a security session. This might result in an extended window for attack by a malicious user who has somehow managed to get access to the session cookies - e.g., by XSS or another attack.

#### Other Terms
Common Session Management, Security service, security session

#### Reason and Prerequisites
Due to a program error in session management, it is possible that a security session might not be invalidated after its expiration.

#### Solution
Update AS Java to the latest patch level of the SP mentioned in the SP Patch Level section.
