### Title
**Unauthorized modification of displayed content in SEM-CPM**

### Description
#### Symptom
FIN-SEM-CPM can be abused by an attacker, allowing them to modify displayed application content without authorization, and to potentially obtain authentication information from other legitimate users.

#### Other Terms
Reflected cross-site scripting, XSS, FIN-SEM-CPM

#### Reason and Prerequisites
Pages within FIN-SEM-CPM do not sufficiently encode OUTPUT parameters, resulting in a reflected cross-site scripting issue. A reflected cross-site scripting attack can be used to non-permanently deface or modify displayed content from a Web site.  
Reflected cross-site scripting can be used to steal another user's authentication information, such as data relating to their current session. An attacker who gains access to this data may use it to impersonate the user and access all information with the same rights as the target user. If an administrator is impersonated, the security of the application may be fully compromised.

#### Solution
Apply the following correction instructions using transaction SNOTE.
