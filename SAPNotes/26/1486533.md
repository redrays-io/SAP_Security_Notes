1486533 - Missing authorization check in RFC call

```markdown
**SAP Security Note 1486533: Missing Authorization Check in RFC Call**

**Type:** SAP Security Note  
**Priority:** Correction with medium priority  
**Released On:** December 14, 2010  
**Component:** Occasional Platform User > Common for all OPU products  
**Version:** 3  
**Status:** Released for Customer

### Description

#### Symptom
An authenticated user can utilize functionality of an RFC call that should have restricted access. This can potentially lead to an escalation of privileges.

#### Reason and Prerequisites
The RFC call lacks proper permission checks for an authenticated user's authorization to access certain functionalities. This may result in undesired system behavior.

### Solution
Implement the correction instructions provided in this note to resolve the issue.

#### Correction Instructions

1. **Create the New Authorization Class and Object:**
   - **Extract the Transport File:**
     - Download and extract the attached file `TransportWithObject_S_OSP.zip` from the [Attachments](https://me.sap.com/sap/support/sapnotes/public/services/attachment.htm?iv_key=012006153200000320532010&iv_version=0003&iv_guid=DB2688C512D6A34D8965DF6FE54C0A88).
     - Copy the transport files to your system's transport folders.
   - **Import the Transport:**
     - Execute transaction `STMS` in your SAP system.
     - Use the Transport Management System to import the objects into your system.

2. **Create Text Message:**
   - Execute transaction `/nSE91`.
   - Enter `/OSP/MSG` as the message class name.
   - Click on `Change`.
   - Add a new entry:
     - **Message Number:** 556
     - **Text:** "User & is not authorized to execute the function module."
   - Save the entry.

3. **Create New Roles with the Authorization Object and Assign Roles to Users:**
   - Log in to the SAP system and execute transaction `/nPFCG`.
   - Enter `S_REMA` in the Role field and click the "Single Role" button.
   - Provide a description, e.g., "Role for REMA user", and save.
   - In the **Authorization** tab, click "Change Authorization Data".
   - Click the "Manually" button in the application toolbar.
   - Enter `S_OSP` as the authorization object and press Enter.
   - Expand all nodes, double-click on the `ACTVT` input field, select activity `16`, and save.
   - Click the `Generate` button and confirm the generation.
   - Assign the role to specific users in the **User** tab and perform a User Comparison.
```
