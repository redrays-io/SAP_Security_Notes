**SAP Security Note 1441953 - Logon Data Can Be Discovered: XSS**

**Summary**
An attacker can exploit a cross-site scripting (XSS) vulnerability in Web Dynpro ABAP to steal the logon data of users. This vulnerability arises from insufficient encoding of input parameters, allowing malicious scripts to execute and potentially compromise user credentials and system security.

**Details**

- **SAP Note/KBA:** 1441953
- **Type:** SAP Security Note
- **Version:** 1
- **Release Status:** Released for Customer
- **Released On:** 11.05.2010
- **Language:** English (Master Language: German)
- **Category:** Program error
- **Priority:** Correction with high priority
- **Component:** Basis Components > Web Dynpro > Web Dynpro ABAP (BC-WD-ABA)

**Symptom**
Certain Web Dynpro ABAP pages do not adequately encode input parameters, resulting in a reflexive XSS vulnerability. This allows attackers to:

- Steal logon data of users.
- Deface or alter the contents of a page.
- Imitate different users within the system, gaining access to their information and authorizations.
- Compromise the security of the entire application if administrative user data is targeted.

**Solution**
Implement the correction instructions provided in this security note or import the relevant Support Package to mitigate the vulnerability.
