## SAP Security Note 604816 - Lockout bypass through function module RFC_SYSTEM_INFO

### Summary
- **Type:** SAP Security Note
- **Version:** 2
- **Priority:** HotNews
- **Status:** Released for Customer
- **Released On:** January 11, 2010
- **Component:** Basis Components > Middleware > RFC

### Description

#### Symptom
The function module `RFC_SYSTEM_INFO` returns information as to whether the used logon data was correct.

#### Other Terms
- **Vulnerabilities**
  - Internet Security Systems: "sap-sapinfo-lockout-bypass"
  - Security Focus Online: "SAP R/3 sapinfo RFC API Account Locking Weakness"
  - "SAP R/3, account locking and RFC SDK"

#### Reason and Prerequisites
The function module `RFC_SYSTEM_INFO` is used in the check of the free resources for load distribution with the asynchronous RFC.

#### Solution
The correction must be made in function module `RFC_SYSTEM_INFO` as described in the [correction instructions](https://me.sap.com/corrins/0000604816/1).

*Note:* If the correction is implemented, a restricted load distribution can occur depending on the scenarios used.

### Available Actions
- [Download for SNOTE](https://notesdownloads.sap.com/note/0040000003091612017)

### Software Components Affected
- **SAP_APPL** from 31I to 31I
- **SAP_APPL** from 40A to 40B
- **SAP_APPL** from 45A to 45B
