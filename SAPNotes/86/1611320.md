SAP Security Note 1611320 - Missing authorization check in CA-GTF-BS-MC

```markdown
**SAP Security Note 1611320** - *Missing authorization check in CA-GTF-BS-MC*

**Type:** SAP Security Note  
**Version:** 1  
**Recency:** New  
**Language:** English  
**Master Language:** German  
**Category:** Program error  
**Priority:** Correction with high priority  
**Status:** Released for Customer  
**Released On:** 08.11.2011  
**Component:** CA-GTF-BS-MC (Cross-Application Components > General Application Functions > Business Application Support > Message Control)

### Symptom
An authenticated user can use functions of CA-GTF-BS-MC to which access should be restricted. This may result in an escalation of privileges.

### Other Terms
Authorization, authorization check, CA-GTF-BS-MC, SD-BF-OC

### Reason and Prerequisites
CA-GTF-BS-MC does not contain authorization checks for checking an authenticated user's authorization to access some of its functions. This may result in undesired system behavior.

### Solution
Carry out the following manual steps that are valid for your release.

#### Valid For
- **Software Component:** SAP_APPL (SAP Application)
  - **Release 31I:** 31I
  - **Release 40B:** 40B
  - **Release 45B:** 45B
  - **Release 46B:** 46B
  - **Release 46C:** 46C
  - **Release 470:** 470
  - **Release 500:** 500
  - **Release 600:** 600
  - **Release 602:** 602
  - **Release 603:** 603
  - **Release 604:** 604
  - **Release 605:** 605
  - **Release 606:** 606

#### Manual Activities

**Manual Activity 1: Valid for Releases 470, 500, 600, 602, 603, 604, 605, 606**
1. Call transaction SE93, enter the transaction code OVNN and choose "Change".
2. In the "Default values for" section, change the transaction to SA39 and set the "Skip initial screen" indicator to 'X'.
3. At the bottom of the page in the default values for the screen fields, delete the blank character before the name RS38M-FUNC_SSET.
4. Save the transaction.

**Manual Activity 2: Valid for Releases 46B, 46C**
1. Call transaction SE93, enter the transaction code OVNN and choose "Change".
2. In the "Default values for" section, change the transaction to SA39 and set the "Skip initial screen" indicator to 'X'. Delete the screen number and the field "From module pool".
3. At the bottom of the page in the default values for the screen fields, delete the blank character before the name RS38M-FUNC_SSET.
4. Save the transaction.

**Manual Activity 3: Valid for Releases 31I, 40B, 45B**
1. Call transaction SE93, enter the transaction code OVNN and choose "Change".
2. In the "Default values for" section, change the transaction to SA39 and set the "Skip initial screen" indicator to 'X'. Delete the screen number and the field "From module pool".
3. Save the transaction.
```
