SAP Security Note 1492070 - Possible disclosure of persistent data in a system

```markdown
**SAP Security Note 1492070**

**Title:** Possible disclosure of persistent data in a system  
**Type:** SAP Security Note  
**Version:** 5  
**Recency:** New  
**Language:** English (Master Language: German)  
**Category:** Program error  
**Priority:** Correction with medium priority  
**Status:** Released for Customer  
**Released On:** 14.02.2012  
**Component:** Service > SAP Solution Manager > Service Data Download (SV-SMG-SDD)

### Description

#### Symptom
A malicious user can exploit an RFC connection to a remote system and use specially crafted inputs to modify database commands, resulting in the retrieval of additional information persisted by the system.

#### Other Terms
SQL injection, database, remote system

#### Reason and Prerequisites
This problem is caused by an SQL injection vulnerability. The code composes an SQL statement that contains strings that can be altered by a malicious user. The manipulated SQL statement can then be used to retrieve data from the database.

#### Solution
Import the relevant Support Package or implement the manual correction instructions.

### Affected Software Components
- **ST-PI 2008_1_46C** and above
- **ST-PI 2008_1_620** and above
- **ST-PI 2008_1_640** and above
- **ST-PI 2008_1_700** and above
- **ST-PI 2008_1_710** and above

### Support Package
- **ST-PI 2008_1_46C:** [SAPKITLRA3](https://me.sap.com/supportpackage/SAPKITLRA3)

### Manual Correction Instructions
Call transaction SE37 for the function modules `/SDF/STUO_GET_ORA_SYS_TABLE` and `/SDF/GET_ORA_SYS_TABLE`. Switch to the "Attributes" tab page and remove the RFC flag by selecting the "Normal Function Module" option.
```
