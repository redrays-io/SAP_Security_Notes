**SAP Security Note 1469368** - *RMA: Authorization check missing in RMA workbench*

**Overview:**
This SAP Security Note addresses a vulnerability in the Retail Method of Accounting (RMA) workbench. Certain functions can be accessed without proper authorization, potentially allowing unauthorized users to escalate their privileges and alter system behavior.

**Details:**

- **Type:** SAP Security Note
- **Priority:** Correction with medium priority
- **Status:** Released for Customer
- **Released On:** September 20, 2010
- **Component:** SAP Business Warehouse > BW only - Business Content and Extractors > BW only - Retail and Consumer Products > BW only - Retail Stock Ledger

**Affected Systems:**
- BI_CONT 7.04
- BI_CONT 7.05

**Issue:**
When the business function `ISR_RETAIL_RMA` is activated in ERP, the RMA workbench in the BW system lacks necessary authorization checks. This omission allows users to call functions without the required permissions, leading to potential unwanted changes in system behavior.

**Solution:**
To resolve this issue, you can either apply the appropriate support packages or implement the provided correction instructions:

1. **Apply Support Packages:**
   - **BI_CONT 7.05:** [Support Package 02](https://me.sap.com/supportpackage/SAPK-70502INBICONT)
   - **BI_CONT 7.04:** [Support Package 08](https://me.sap.com/supportpackage/SAPK-70408INBICONT)

2. **Implement Correction Instructions:**
   - [Correction Instructions](https://me.sap.com/corrins/0001469368/384)

Additionally, define a secure RFC connection for synchronous method calls in ALE Customizing by following the Implementation Guide (IMG):
- [SAP NetWeaver Application Server → Application Server → IDoc Interface / ALE → Communication → Determine RFC Destinations for Method Calls](https://me.sap.com/notes/0001469368/D)
