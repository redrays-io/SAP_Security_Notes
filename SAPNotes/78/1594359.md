## SAP Security Note 1594359 - FI: Potential Directory Traversal - Mexico

### **Summary**
This security note addresses potential directory traversal vulnerabilities in the **XX-CSC-MX** component of the FI module. Malicious users may exploit these vulnerabilities to read or write arbitrary files on the remote server, risking the disclosure of confidential information or the corruption of data and system behavior.

### **Details**

#### **Symptom**
- Potential Directory Traversal in the following components: **XX-CSC-MX**.

#### **Reason and Prerequisites**
- The programs contained in the correction instructions have vulnerabilities that allow malicious users to read arbitrary files on the remote server, potentially disclosing confidential information.
- Some programs also allow writing arbitrary files on the remote server, which can lead to data corruption or altered system behavior.

#### **Solution**
Refer to [SAP Note 1497003](https://me.sap.com/notes/1497003) for additional information and instructions. Implementing the corrections from this note is a prerequisite for applying this security note.

**Logical File Names Used in this Solution:**
- `FI_RFIDMXFORMAT29_FILE`
- `FI_RFIDMXCTOP_FILE`
- `FI_RFIDMXVTOP_FILE`

**Logical File Path Used:**
- `FI_MX_FILE_PATH`

The applications `RFIDMXFORMAT29`, `RFIDMXCTOP`, and `RFIDMXVTOP` pass an additional parameter (`parameter_1` - `sy-cprog`) to the Function Module `FILE_VALIDATE_NAME`. This allows customers to configure the physical paths for the logical file name `FI_MX_FILE_PATH`.
