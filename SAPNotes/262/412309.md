### Title
**412309 - Authorization profile for RFC user for IPC**

### Description
#### Symptom
You want to create a user for the communication between the Internet Pricing and Configurator (IPC) and an ABAP system. When you do this, you do not want to assign too many (and unnecessary) authorizations to this user, as would be the case with SAP_ALL/SAP_NEW, for example.

#### Other Terms
Internet Pricing and Configurator, IPC, installation, administration, user, authorization  
Transaction SU01, PFCG

#### Reason and Prerequisites
You have installed an IPC and want to create a user for it as specified in the installation guide that enables the IPC to communicate with the database of an ABAP system.

#### Solution
There are the following different scenarios:

- **Scenario A:**
  You want the IPC to communicate with a CRM or SRM system. This also applies to Internet Sales (ISA).

- **Scenario B:**
  You want the IPC to communicate with an R/3 system. This also applies to the Dataloader.

- **Scenario C:**
  The IPC runs in the Virtual Machine Container (VMC). This applies as of CRM 5.0 or SRM 5.0.

**Proceed as follows:**

1. Create a new role that contains only the authorization object `S_RFC` with the following values:

   - **RFC_TYPE:** `'FUGR'`
   - **Scenario A:**
     - **RFC_NAME:** `'SYST, RFC1, COM_SCE_SV, COM_IPC_SV, IPC_V01, COM_PME_ENQUEUE, COM_PME_ID_SERVER, COM_PME_DB_WRITE'`
   - **Scenario B:**
     - **RFC_NAME:** `'SYST, RFC1, CRM0, CFIL, SDTX, ORFC'`
   - **Scenario C:**
     - **RFC_NAME:** `'SYST, RFC1, IPC_V01, CNV_CURRENCY, CNV_TEXT, CNV_UNITS, /SAPCND/DET_CONDITIONS, SPC_*, CA_TTE_VMC_RFC, COM_CFG_API_*, COM_PRODUCT, IPC_EXCHANGE_COMMANDS, SDIFRUNTIME, SYSU'`
     - **ACTVT:** `'16'`
     - *(You must enter the authorization object manually.)*

   **Function groups required:**

   - **SYST** for remote login (all scenarios)
   - **RFC1** for JCo (all scenarios)
   - **COM_SCE_SV** for SCE (scenario A)
   - **COM_IPC_SV** for SCE (scenario A)
   - **IPC_V01** for SPE, SCE, and TTE (scenarios A+C)
   - **COM_PME_ENQUEUE** for PME (scenario A)
   - **COM_PME_ID_SERVER** for PME (scenario A)
   - **COM_PME_DB_WRITE** for PME (scenario A)
   - **CRM0** for Dataloader (scenario B)
   - **CFIL** for Dataloader (scenario B)
   - **SDTX** for Dataloader (scenario B)
   - **ORFC** for Dataloader (scenario B)
   - **CNV_CURRENCY** for SPE (scenario C)
   - **CNV_TEXT** for SPE (scenario C)
   - **CNV_UNITS** for SPE (scenario C)
   - **/SAPCND/DET_CONDITIONS** for SPE (scenario C)
   - **SPC_*** for SPE (scenario C)
     - Placeholder for SPC_DEPRECATED
     - SPC_DEPRECATED_PERF
     - SPC_LEAN_DOCUMENT
   - **CA_TTE_VMC_RFC** for TTE (scenario C)
   - **COM_CFG_API_*** for SCE (scenario C)
     - Placeholder for COM_CFG_API_UI_COMMAND
     - COM_CFG_API_ENGINE_COMMAND
     - COM_CFG_API_MASTERDATA
   - **SYSU** for SCE (scenario C)
   - **COM_PRODUCT** for SCE (scenario C)
   - **IPC_EXCHANGE_COMMANDS** for SCE (scenario C)
   - **SDIFRUNTIME** for SCE (scenario C)

   Additionally, if using **Scenario C**, add the authorization object `COM_PRD` with activity `'03'` (Display).

2. Create users of the "System" type with this role only.

   This user cannot log on in the dialog but only by Remote Function Call (RFC), and is authorized to call any RFC-enabled function modules that do not perform further authorization checks. This setup is required by the IPC.
