SAP Security Note 1368764 - Authorization Check Missing in AM-CMS and AO-CMS Services

```markdown
**Type:** SAP Security Note  
**Version:** 4  
**Priority:** Correction with High Priority  
**Status:** Released for Customer  
**Released On:** September 13, 2011  
**Component:** Financial Services > Collateral Management

---

### Symptom

An unauthenticated user can use functions of the Collateral Management System (CMS) to which access should be restricted. This may result in an escalation of privileges.

### Other Terms

Authorization, authorization check, Collateral Management System, AO-CMS, AM-CMS

### Reason and Prerequisites

Collateral Management System does not contain authorization checks for verifying an unauthenticated user's authorization to access certain functions. This may result in undesired system behavior.

### Solution

Apply the [correction instruction](https://me.sap.com/corrins/0001368764/1534) or import the [support package SAPKISC309](https://me.sap.com/supportpackage/SAPKISC309).

### Manual Activities

Manually adjust message classes using transaction SE91 as described in the Correction Instructions:

1. Enter transaction `SE91`.
2. Enter the message class `CMS_ANL_PRES_CALC`.
3. In change mode for Message number `005`, enter the following text:
   ```
   'Constellation cannot be displayed; missing authorization for admin org &'.
   ```
4. Check the self-explanatory flag.
5. Save and exit.
6. Repeat the steps for message class `SERVICES_CMS` and Message number `080` with the text:
   ```
   'Missing authorization for admin org &'.
   ```
7. Check the self-explanatory flag, then save and exit.

### Preconditions

Ensure the following SAP Notes are applied:

- [1109760 - Collateral Constellation Create service failed](https://me.sap.com/notes/1109760)
- [1311573 - CRM CMS integration scenario problem with business partner](https://me.sap.com/notes/1311573)
```
