**SAP Security Note 2026174** addresses a critical vulnerability in Apache Struts1.x (CVE-2014-0094). Below are the essential details:

### Symptom

**UPDATE 24th November**: This note has been re-released with updated **CVSS details and Solution** information.

The `ParametersInterceptor` in Apache Struts versions earlier than 2.3.16.3 allows remote attackers to manipulate the `ClassLoader` via the `class` parameter, which is passed to the `getClass` method. This vulnerability can be exploited to manipulate resources used to serve BI Launchpad, LCM, and Monitoring.

### Other Terms

- **CVE**: CVE-2014-0094
- **Component**: BI-BIP-INV

### Reason and Preconditions

An attacker can infiltrate the application through vulnerabilities in the Struts framework, allowing malicious code injection to consume resources unnecessarily.

### Solution

This vulnerability has been fixed by modifying configurations related to the Struts Framework. The fix has been delivered with the following updates:

- **SAP BusinessObjects Enterprise XI3.1**: Refer to the “[Hotfix for XI3.1 FixPack 6.4](https://me.sap.com/servicessupport)” on the SAP Service Market Place.

- **SAP BusinessObjects Enterprise XI4.0 SP10**
- **SAP BusinessObjects Enterprise XI4.1** FP1.11, FP2.9, FP3.4, FP4.1
