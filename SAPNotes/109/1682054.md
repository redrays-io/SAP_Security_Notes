**SAP Security Note 1682054 - Unauthorized Modification of Displayed Content in SRM-SUS**

**Summary**
A vulnerability has been identified in the SAP Supplier Relationship Management - Supplier Self-Services (SRM-SUS) component that allows unauthorized modification of displayed application content. This can potentially enable malicious users to obtain authentication information from other legitimate users through Cross Site Scripting (XSS) attacks.

**Details**
- **Type:** SAP Security Note
- **Version:** 2
- **Recency:** New
- **Language:** English
- **Category:** Program error
- **Priority:** Correction with high priority
- **Status:** Released for Customer
- **Released On:** May 8, 2012
- **Component:** Supplier Relationship Management > Supplier Self-Services (SRM-SUS)
- **Affected Software Component Version:** SRM_SERVER 702
- **Support Package:** [SAPK-70203INSRMSRV](https://me.sap.com/supportpackage/SAPK-70203INSRMSRV)

**Vulnerability Details**
The BSP applications using an extension tag rely on proper encoding. Missing or improper encoding within an extension may lead to XSS vulnerabilities in the BSP applications using the extension. Specifically, the BSP tags of the SRMSUS extension might be exploited due to missing encoding of all attribute values passed to the extension.

**Prerequisites**
Before applying this security note, ensure that [SAP Note 1582870](https://me.sap.com/notes/1582870) and [SAP Note 1582867](https://me.sap.com/notes/1582867) have been implemented via SNOTE as prerequisites.

**Solution**
Apply SAP Security Note 1682054 using SNOTE or import the changes through the relevant support package. This will address the unauthorized modification vulnerability in the SRM-SUS component.
