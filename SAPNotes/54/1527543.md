## Description

### Symptom
The component `WEBREQ_REQ_DISP` can be abused by a malicious user allowing them to modify displayed application content without authorization and to potentially obtain authentication information from other legitimate users.

### Other Terms
Reflected Cross Site Scripting, XSS, WEBREQ_REQ_DISP

### Reason and Prerequisites
Pages within the component `WEBREQ_REQ_DISP` do not sufficiently encode output parameters, resulting in a reflected cross site scripting issue. A reflected cross site scripting attack can be used to non-permanently deface or modify displayed content from a website.

Reflected cross site scripting can be used to steal another user's authentication information such as data relating to their current session. An attacker who gains access to this data may use this to impersonate the user and access all information with the same rights as the target user. In case of an administrator being impersonated, a full compromise of the application's security may occur.

**Manual Activities:**
You have to perform this manual post-implementation step manually and separately in each system **after** you have imported the Note to implement.

For the class `CL_WEBREQ_DISP_ITERATOR`, some text elements must be created.

1. Call transaction SE24 and enter class name `CL_WEBREQ_DISP_ITERATOR`. Press the "Display" button.
2. Go via menu -> "Goto" -> "Text elements" and press the "Display ↔ Change" icon.
