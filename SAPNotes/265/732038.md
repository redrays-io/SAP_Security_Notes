### Title

**732038 - Password history: Oldest entry is not checked**

### Description

#### Symptom

Contrary to the description in note 2467, the password history only contains 4 instead of 5 entries. This means that the user can reuse an old password after having chosen 4 (instead of 5) user-defined passwords.

#### Other Terms

Password, history, USR02-BCOD5

#### Reason and Prerequisites

This is an error in the kernel: The system does not check the last (that is, oldest) entry.

#### Solution

Import a new kernel:

- **For Basis Release 4.6x:** Import kernel 4.6D as of patch number 1825.
- **For Basis Release 6.10/6.20:** Import kernel 6.20 as of patch number 1461.
- **For Basis Release 6.40:** Import kernel 6.40 as of patch number 15.
